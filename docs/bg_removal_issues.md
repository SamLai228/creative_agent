# 去背功能痛點紀錄

## 問題描述

`assets/0-18.png` 是一張 3D 插畫全家福，人物周圍有浮動的綠色線稿裝飾圖示（蘑菇、奶瓶、護目鏡）。
使用 rembg + birefnet-general 去背後，這些小圖示沒有被移除，仍殘留在透明 PNG 中。

---

## 根本原因

**birefnet-general 是語意顯著性分割模型**，它找的是「畫面中的顯著前景物件」，而非單純的「人物」。
這張插畫的設計意圖本來就是把小圖示當作整體構圖的一部分，模型因此把它們也判為前景。

此外，rembg 的 alpha matting 後處理會在前景邊緣補半透明像素。結果是：
即使視覺上圖示和人物之間有空白，alpha channel 中仍存在肉眼不可見的半透明橋接像素，把圖示連接到主體。

---

## 嘗試過的方法

### 方法一：色彩閾值還原（`_restore_colored_pixels`）

**思路：** 白色背景圖中，所有「有色像素」都是前景。rembg 跑完後，補回原圖中有色但被移除的像素。

**為什麼沒效：** 方向反了。這個方法是為了「保留」被誤移除的有色像素，但使用者需求是「移除」多餘的圖示，結果反而更糟。

---

### 方法二：最大連通區域（erode_px=20）

**思路：** rembg 後對 alpha mask 做侵蝕（20px），斷開細橋，找最大連通區域保留，其餘清空，再膨脹回來。

**測試結果：**
```
erode=20: components=8, largest=89604, total_fg=236286
```

**為什麼沒效：** 20px 侵蝕不夠斷開橋接。護目鏡圖示和男生頭髮之間的半透明橋在 alpha>128 的二值 mask 下仍保持連通，屬於同一個 component。

---

### 方法三：最大連通區域（erode_px=40）

**思路：** 加大侵蝕到 40px，再膨脹 45px（40+5）還原。

**測試結果：**
```
erode=40: components=2
  component 1: size=9044, bbox rows 84~186, cols 496~631  ← 圖示
  component 2: size=36161, bbox rows 183~484, cols 325~645 ← 主體
```

40px 侵蝕後確實分成 2 個獨立 component。

**為什麼沒效：** 膨脹量只有 45px，不足以把侵蝕後的主體還原回原始大小。男生頭頂（頭髮細長）在 40px 侵蝕後和身體分離，膨脹後無法重新連回——導致頭部消失。

---

### 方法四：侵蝕找主體 + 大幅膨脹 + 原始 alpha 取交集（erode_px=40, dilate=120px）

**思路：** 侵蝕找到主體的侵蝕核（component 2），膨脹 120px（40×3）覆蓋整個原始主體，再與原始 alpha 取交集，理論上能精確還原邊緣。

**為什麼沒效：** 膨脹 120px 後，主體的邊界向上擴張超過 120 行，覆蓋到了圖示所在的行（rows 84~186）。與原始 alpha 取交集後，圖示的 alpha>128 像素仍被保留。

---

## 核心困境

birefnet 的分割結果中，圖示區域（rows 0~200）的 alpha 像素數量：
```
pixels > 128: 48,646 個（佔 top 200 rows 的約 24%）
```

這些像素大量且稠密，即使採用連通分析，因為圖示面積大、alpha 值高，任何基於膨脹的「從主體往外擴」策略都難以避免把圖示區域包含進去。

---

## 未來可行方向（未實作）

1. **換模型**：改用 `birefnet-portrait` 或 `u2net_human_seg`，這類模型專門針對人物，理論上不會把裝飾圖示判為前景。

2. **手動遮罩**：在原圖用塗黑方式標記「一定是背景」的區域（即圖示所在位置），再送進 rembg。

3. **兩段去背**：先跑一次 rembg 取得人物 bbox，裁切後只對 bbox 內的區域做精細去背，bbox 外自動清空。

4. **SAM（Segment Anything）**：用 SAM 點擊人物位置做 prompted segmentation，只分割指定點附近的物件，完全避開圖示。
